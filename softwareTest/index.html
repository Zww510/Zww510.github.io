信息化知识
控制论的创始人 维纳 认为：信息就是信息，既不是物质也不是能量。
信息论的创始人 香农 认为：信息就是能够用来消除不确定的东西。
信息的传输模型
信源 -> 编码 -> 信道 （噪声） -> 解码 -> 信宿
  • 信源：产生信息的实体，信息产生后，由这个实体向外传播。
  • 信宿：信息的归宿或接受者。
  • 信道：传送信息的通道，如 TCP/IP，光纤，铜轴电缆，双绞线，4G/5G网络，卫星或微波等。
  • 编译器：如量化器，压缩编码器，调制器等，使信源输入的信号转换成合适与信道传输的信号。
  • 译码器：将接受的信号转换成信宿能理解的信号，如解调器，译码器，数模转换器等。
  • 噪声：可以理解为干扰，当噪声携带的信息达到一定程度后，可将信道中传输的信息淹没从而导致传输失败。
信息的质量属性
1. 精确性：对事物状态描述的精确程度
2. 完整性：对事物状态描述的全面程度
3. 可靠性：指信息的来源，采集方法，传输过程是可以信任的，符合预期
4. 及时性：指获得信息的时刻与事假发生时刻的间隔长短
5. 经济性：信息获取，传输带来的成本
6. 可验证性：信息主要质量属性可以被证实或证伪的程度
7. 安全性：信息的生命周期中，信息可以被非授权访问的可能性，可能性越低，安全性越高
系统的基本概念
系统具有以下几个特点
1. 目的性：定义一个系统，组成一个系统或者抽象出一个系统，都有明确的目标或者目的，目标决定了系统的功能
2. 可嵌套性：系统可以包含若干子系统
3. 稳定性：受规则约束，系统的内部结构和秩序是可预见的。稳定性强的系统使得系统在受到外部作用的同时，内部结构和秩序仍然能够保持
4. 开放性：是指系统的可访问性，决定了系统可以被外部环境识别
5. 脆弱性：系统可能丧失结构，功能，秩序的特性，这个特性往往是隐藏不易被外界感知的
6. 健壮性：系统面临干扰，输入错误，入侵等因素，系统可能出现非预期的状态而丧失原有功能，系统具有能够抵御出现非预期状态的特性称为健壮性

电子商务应该具有以下基本特征：
电子商务的发展历程
1. 原始电子商务：凡使用了诸如电报，电话，广播，电视，传值及计算机，计算机网络等手段，工具和技术进行商务活动，都可以称之为电子商务

EDI （电子数据交换），是连接原始电子商务和现代电子商务的纽带

面向对象的基本概念
1. 对象
  • 由数据及其操作所构成的封装体，是系统中用来描述客观事物的一个模块，是构成系统的基本单位，三个基本要素，分别是 对象标识，对象状态和对象行为
2. 类
  • 现实世界中实体的形式化描述，类将该实体熟悉（数据）和操作（函数）封装在一起
    • 类和对象的关系可以理解为，对象是类的实例，类是对象的模板
3. 抽象：通过特定的实例 抽取共同特征 以后形成概念的过程
4. 封装：将相关的概念组成一个单元模块，并通过一个名称来引用它
5. 继承：表示 类 之间的层次关系（父类与子类），这种关系使的其某类对象可以继承另外一类对象的特征
6. 多态：使得在 多个类中可以定义同一个 操作或属性名，并在某个类中可以又不同的实现
7. 接口：描述对操作规范的说明
8. 消息：体现对象间的交互通过它可以向目标对象发送操作请求
9. 组件：表示软件系统可替换，物理的组件部分，封装了模块功能的实现
10. 复用：指将已有的软件及其有效成分用于构造新的软件或系统
11. 模式：描述了一个不断重复发生的问题，以及该问题的解决方案

计算机网络知识
网络技术标准与协议： OSI七层协议
1. 物理层：该层包括物理联网媒介，如电缆连线连接器，具体标准有（RS232, V.35, RJ45, FDDI）
2. 数据链路层：它控制的是网络层与物理层之间的通信。常见的协议有（IEEE802.3/.2, HDLC, PPP, ATM）
3. 网络层：主要功能是将网络地址（IP地址）翻译成对应的物理地址，例如 网卡地址，并决定如何将数据从发送方路由到接受发。具体协议（IP, ICMP, IGMP, IPX, APP）
4. 传输层：主要负责确保数据可靠，顺序，无错的从A点到传到B点。具体协议（TCP, UDP, SPX）
5. 会话层：负责在网络中的两个节点之间建立和维护通信，以及提供交互会话的管理的功能。常见协议（RPC, SQL, NFS）
6. 表示层：应用程序和网络之间的翻译官，数据将按照网络能理解的方式进行格式化。常见协议（JPEG, ASCLL, GIF, DES, MPEG）
7. 应用层：负责对软件提供接口以使程序能使用网络服务，如事物处理程序，文件传送协议和网络管理等。常见协议（HTTP, TELNET, FTP, SMTP, HTTPS）
网络存储技术
1. 直连式存储 DNS：通过标准接口（SCSI）连接存储器与服务器
2. 网络连接存储 NAS：通过网络连接，即插既用
3. 存储区域网络 SAN：通过高速光纤连接，采用块（block）级别存储，成本较高，技术较复杂
网络安全--信息安全的基本要素
1. 机密性：确保信息不暴露给未授权的实体或进程
2. 完整性：只有得到允许的人才能修改数据，并且能够判别出数据是否已篡改
3. 可用性：得到授权的实体在需要时可访问数据，既攻击者不能占用所有的资源而阻碍授权者的工作
4. 可控性：可以控制授权范围内的信息流向及行为方式
5. 可审查性：对出现的网络安全问题提供调查的依据手段
网络安全--网络和信息安全产品
1. 防火墙：通常比喻为网络安全的大门，用来鉴别什么样的数据包可以进出企业内部网。传统防火墙无法阻止和检测基于数据内容的黑客攻击和病毒入侵，同时也无法控制内部网络之间的违规行为
2. 扫描器：是入侵检测的一种，主要用来发现网络服务，网络设备和主机的漏洞，通过定期的检测与比较，发现入侵和违规行为留下来的痕迹，无法发现正在入侵的违规行为，而且它还有可能变成攻击者的工具
3. 防毒软件：可以检测，清除各种文件类型病毒，宏病毒和邮件病毒等等
4. 安全审计系统：通过独立的，对网络行为和主机操作提供全面与忠实的记录，方便用户分析与审查事故原因，就像飞机上的黑匣子

架构

三点估算
  • 最乐观的时间（TO），最可能的时间（TM），最悲观的时间（TP）
  • 活动历史的均值 Te = （To + 4Tm + Tp）/ 6。ps：4Tm = 4倍的Tm
  • 标准差 o = （Tp - To） / 6
总浮动时间：左下角减去左上角或右下角减去右上角（最迟完成时间减去最早完成时间）
自由浮动时间：紧后活动的最早开始时间减去最早结束时间，就是左上角减去右上角
挣值管理（EVM）是把范围，进度和资源绩效综合起来考虑，以评估项目绩效和进展方法
三个参数
  • 计划值（PV）：截至某一时刻起，应该完成的 / 可以完成的工作量的预算值
  • 实际成本（AC）：截至某一时刻起，实际的花费
  • 挣值（EV）：截至某一时刻起，已完成的 / 实际完成的工作量的预算值
4个指标
进度偏差 SV = EV - PV
  • >0，进度超前
  • < 0，进度落后
成本偏差 CV = EV - AC
  • >0，成本节约
  • < 0，成本超支
进度绩效指标 SPI = EV / PV
  • >1，进度超前
  • < 1，进度落后
成本绩效指数 CPI = EV / AC
  • >1，成本节约
  • < 1，成本超支
完工预算 BAC：所有计划值 PV 之和
完工尚需估算 ETC：剩余工作量.所需的成本
  • 经过纠正后；问题不在发生；非典型；ETC = BAC - EV
  • 不纠正；问题持续存在；典型；ETC = (BAC - EV) / CPI
完工估算 EAC：重新估算项目所需要的成本
  • 通用公式：EAC = AC + ETC
  • 通过纠正后；问题不在发生；非典型；EAC = AC + BAC - EV
  • 不纠正；问题持续存在；典型；EAC = BAC / CPI
完工偏差 VAC：完工预算 BAC 与完工估算 EAC 的差值；VAC = BAC - EAC

七种质量管理工具：流 - 因 - 直 - 点 - 累 - 制 - 查
  • 流程图 - 因果图 - 直方图 - 散点图 - 帕累托图 - 控制图 - 检查表
亲和图：用来对大量创意进行分组的技术，以便进一步审查和分析，可以对潜在缺陷进行分类，显示最应该关注的区域
鱼骨图（又称：因果图、石川图）：将问题分解为离散的分支，有助于识别问题的主要原因和根本原因
控制图：用来确定一个过程是否稳定，是否具有可预测的绩效
直方图：一种展示数字数据的条形图，可以展示每个可交付成果的缺陷数量，缺陷成因的排列，各个过程的不合格次数
矩阵图：在行列交叉的位置展示因素，原因，和目标之间的关系强弱，有助于识别对项目成功至关重要的质量测量指标
散点图（又称：相关图）：展示两个变量之间关系的图形
帕累托图：识别造成大多数问题的少数原因
