信息化知识
控制论的创始人 维纳 认为：信息就是信息，既不是物质也不是能量。
信息论的创始人 香农 认为：信息就是能够用来消除不确定的东西。
信息的传输模型
信源 -> 编码 -> 信道 （噪声） -> 解码 -> 信宿
  • 信源：产生信息的实体，信息产生后，由这个实体向外传播。
  • 信宿：信息的归宿或接受者。
  • 信道：传送信息的通道，如 TCP/IP，光纤，铜轴电缆，双绞线，4G/5G网络，卫星或微波等。
  • 编译器：如量化器，压缩编码器，调制器等，使信源输入的信号转换成合适与信道传输的信号。
  • 译码器：将接受的信号转换成信宿能理解的信号，如解调器，译码器，数模转换器等。
  • 噪声：可以理解为干扰，当噪声携带的信息达到一定程度后，可将信道中传输的信息淹没从而导致传输失败。
信息的质量属性
1. 精确性：对事物状态描述的精确程度
2. 完整性：对事物状态描述的全面程度
3. 可靠性：指信息的来源，采集方法，传输过程是可以信任的，符合预期
4. 及时性：指获得信息的时刻与事假发生时刻的间隔长短
5. 经济性：信息获取，传输带来的成本
6. 可验证性：信息主要质量属性可以被证实或证伪的程度
7. 安全性：信息的生命周期中，信息可以被非授权访问的可能性，可能性越低，安全性越高
系统的基本概念
系统具有以下几个特点
1. 目的性：定义一个系统，组成一个系统或者抽象出一个系统，都有明确的目标或者目的，目标决定了系统的功能
2. 可嵌套性：系统可以包含若干子系统
3. 稳定性：受规则约束，系统的内部结构和秩序是可预见的。稳定性强的系统使得系统在受到外部作用的同时，内部结构和秩序仍然能够保持
4. 开放性：是指系统的可访问性，决定了系统可以被外部环境识别
5. 脆弱性：系统可能丧失结构，功能，秩序的特性，这个特性往往是隐藏不易被外界感知的
6. 健壮性：系统面临干扰，输入错误，入侵等因素，系统可能出现非预期的状态而丧失原有功能，系统具有能够抵御出现非预期状态的特性称为健壮性
信息化
信息化是推动经济社会发展转型的一个历史过程。信息化从小到大分成五个层次：
1. 产品信息化：产品信息化是信息化的基础
2. 企业信息化：是指企业在产品的设计，开发，生产，管理，经营等多个环节中广泛的利用信息技术，ERP,CRM,SCM(供应链管理)等
3. 产业信息化：指农业，交通运输业，工业等传统产业广泛的利用信息技术，进一步提高生产水平
4. 国民经济信息化：金融，贸易，投资，计划，营销等组成一个信息大系统
5. 社会生活信息化：商务，教育，政务，交通等在内的整个社会体系采用的先进信息技术，如智慧城市，物联网金融
国家信息化体系要素
我国陆续建成了以 "两网，一站，四库，十二金" 工程为代表的国家信息系统。"十二五"期间，"智慧城市" 做为新兴的国家级战略规划
  • 两网：是指政务内网和政务外网
  • 一站：是指政府门户网站，一般是指 G2C
  • 四库：即建立人口，法人单位，空间地理和自然资源，宏观经济等四个基础的数据库
  • 十二金：12个重点业务系统
1. 信息技术应用--龙头，主阵地
2. 信息技术和产业--信息化建设的基础
3. 信息化人才--成功之本
4. 信息化政策法规和标准规范--根本保障
5. 信息资源--核心任务
6. 信息网络--信息技术应用的基础
口诀：鹰头 - 鸡基 - 人本 - 法宝 - 自信 - 网基
国家信息化战略和规划
基本原则
  • 1. 统筹发展，有序推进
  • 2. 需求牵引，市场导向
  • 3. 完善机制，创新驱动
  • 4. 加强管理，保障安全
企业信息化概述
企业信息化结构
  • 产品（服务）层
  • 作业层
  • 管理层
  • 决策层
口诀：产业观测
企业信息化发展的战略要点
1. 以信息化带动工业化
2. 信息化与企业业务全过程的融合，渗透
3. 信息产业发展与企业信息化良性互动
4. 充分发挥政府的引导作用
5. 高度重视信息安全
6. 企业信息化与企业的改组改造和形成现代企业制度的有机结合
企业资源计划ERP
ERP的概念：随着计算机技术和管理理论的发展，在MRP引入 质量管理，设备管理，财务管理和人力资源管理，形成统一的企业资源计划，即 ERP。
ERP是一个以财务会计为核心的信息系统，对采购，生产，成本，库存，销售，运输，财务和人力资源 等进行规划和优化，使企业利润最大化。
ERP系统的功能
  • 财务管理
  • 生产控制管理
  • 物流管理（销售管理，库存控制，采购管理）
  • 人力资源管理
客户管理管理CRM
客户数据
1. 描述性数据：客户的基本信息
2. 促销性数据：企业曾经为客户提供的产品和服务的历史数据
3. 交易性数据：历史购买记录数据，投诉数据，请求提供咨询及其他服务的相关数据
功能模块
1. 自动化的销售
2. 自动化的市场营销
3. 自动化的客户服务

1. 政府间的电子政务（G2G）
2. 政府对企业的电子政务（G2B）
3. 政府对公众的电子政务（G2C）

电子商务
电子商务应该具有以下基本特征：
电子商务的发展历程
1. 原始电子商务：凡使用了诸如电报，电话，广播，电视，传值及计算机，计算机网络等手段，工具和技术进行商务活动，都可以称之为电子商务
2. 现代电子商务：在网络环境下，买卖双方不需要见面，实现线上交易，在线支付，智能配送以及相关综合服务的一切活动
EDI （电子数据交换），是连接原始电子商务和现代电子商务的纽带
  • 普遍性：电子商务作为一种新型的交易方式，将生产企业，流通企业，消费者以及金融企业和监管者集成到了数字化的网络经济中
  • 便利性：不受地域，环境，交易时间的限制
  • 整体性：保证了交易过程的规范和严谨
  • 安全性：采取加密，身份认证，防入侵，电子签名，防病毒等技术手段确保交易活动的安全性
  • 协调性：企业和客户之间，企业和企业之间，客户和金融之间需要有序的协作，共同配合来完成交易
电子商务的类型
  • 企业与企业之间的电子商务（B2B）
  • 商业企业与消费者之间的电子商务（B2C）
  • 消费者与消费者之间的电子商务（C2C）
电子商务与线下实体店有机结合像消费者提供商品和服务，称为（020）模式
商业智能（BI）
商业智能是数据仓库，OLAP和数据挖掘等技术的综合运用
主要功能
  • 1. 数据仓库
  • 2. 数据ETL
  • 3. 数据统计输出（报表）
  • 4. 分析功能
三个层次
  • 1. 数据报表
  • 2. 多维数据分析
  • 3. 数据挖掘：是从特定形式的数据集中提炼的知识工程
OLAP 和 OLTP
OLTP：主要用于基本的，日常的事物处理，例如银行交易
OLAP：支持复杂的分析操作，侧重决策支持，并且提供直观易懂的查询结果，也被称为多维分析
实施商业智能的步骤
1. 需求分析
2. 数据仓库建模
3. 数据抽取
4. 建立商户智能分析报表
5. 用户培训和数据模拟测试
6. 系统改进和完善
智慧城市
功能层分为以下：
物联感知层：信息采集设备，各类传感器，监控摄像头，GPS终端，FIFID标签
通信网络层：互联网，电信网，广播电视网以及传输介质为光纤的城市专用网
计算与存储层：软件资源，计算资源和存储资源
数据及服务支撑层：SOA，智能挖掘分析，云计算，大数据
智慧应用层：智慧医疗，智慧交通，智慧家政，智慧园区，智慧社区，智慧旅游等
支撑体系分为以下：
安全保障体系
建设和运营管理体系
