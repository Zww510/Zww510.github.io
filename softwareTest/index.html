信息化知识
控制论的创始人 维纳 认为：信息就是信息，既不是物质也不是能量。
信息论的创始人 香农 认为：信息就是能够用来消除不确定的东西。
信息的传输模型
信源 -> 编码 -> 信道 （噪声） -> 解码 -> 信宿
  • 信源：产生信息的实体，信息产生后，由这个实体向外传播。
  • 信宿：信息的归宿或接受者。
  • 信道：传送信息的通道，如 TCP/IP，光纤，铜轴电缆，双绞线，4G/5G网络，卫星或微波等。
  • 编译器：如量化器，压缩编码器，调制器等，使信源输入的信号转换成合适与信道传输的信号。
  • 译码器：将接受的信号转换成信宿能理解的信号，如解调器，译码器，数模转换器等。
  • 噪声：可以理解为干扰，当噪声携带的信息达到一定程度后，可将信道中传输的信息淹没从而导致传输失败。
信息的质量属性
口诀：精完可及经验安
1. 精确性：对事物状态描述的精确程度
2. 完整性：对事物状态描述的全面程度
3. 可靠性：指信息的来源，采集方法，传输过程是可以信任的，符合预期
4. 及时性：指获得信息的时刻与事件发生时刻的间隔长短
5. 经济性：指获取信息带来的成本在可接受的范围之内
6. 可验证性：信息主要质量属性可以被证实或证伪的程度
7. 安全性：信息的生命周期中，信息可以被非授权访问的可能性，可能性越低，安全性越高
系统的基本概念
系统具有以下几个特点
1. 目的性：定义一个系统，组成一个系统或者抽象出一个系统，都有明确的目标或者目的，目标决定了系统的功能
2. 可嵌套性：系统可以包含若干子系统
3. 稳定性：受规则约束，系统的内部结构和秩序是可预见的。稳定性强的系统使得系统在受到外部作用的同时，内部结构和秩序仍然能够保持
4. 开放性：是指系统的可访问性，决定了系统可以被外部环境识别
5. 脆弱性：系统可能丧失结构，功能，秩序的特性，这个特性往往是隐藏不易被外界感知的
6. 健壮性：系统面临干扰，输入错误，入侵等因素，系统可能出现非预期的状态而丧失原有功能，系统具有能够抵御出现非预期状态的特性称为健壮性

面向对象的基本概念
1. 对象
  • 由数据及其操作所构成的封装体，是系统中用来描述客观事物的一个模块，是构成系统的基本单位，三个基本要素，分别是 对象标识，对象状态和对象行为
2. 类
  • 现实世界中实体的形式化描述，类将该实体属性（数据）和操作（函数）封装在一起
    • 类和对象的关系可以理解为，对象是类的实例，类是对象的模板
3. 抽象：通过特定的实例 抽取共同特征 以后形成概念的过程
4. 封装：将相关的概念组成一个单元模块，并通过一个名称来引用它
5. 继承：表示 类 之间的层次关系（父类与子类），这种关系使的其某类对象可以继承另外一类对象的特征
6. 多态：使得在 多个类中可以定义同一个 操作或属性名，并在某个类中可以有不同的实现
7. 接口：描述对操作规范的说明
8. 消息：体现对象间的交互，通过它可以向目标对象发送操作请求
9. 组件：表示软件系统可替换，物理的组件部分，封装了模块功能的实现
10. 复用：指将已有的软件及其有效成分用于构造新的软件或系统
11. 模式：描述了一个不断重复发生的问题，以及该问题的解决方案
中间件
中间件技术：是位于硬件，操作系统等平台和应用之间的通用服务。借助中间件，解决分布系统的异构问题
中间件分类：
1. 数据库访问中间件：通过一个抽象层访问数据库，从而允许使用相同或相似的代码访问不同的数据库资源。ODBS，JDBC
2. 远程过程调用中间件（RPC）：是一种分布式应用程序的处理方法。Sutp，Skeleton
3. 面向消息中间件（MOM）：利用高效可靠的消息传递机制，进行平台无关的数据传递，还可以基于数据通信进行分布系统的集成。IBM的MQSeries
4. 分布式对象消息中间件：建立对象之间 客户/服务器 关系的中间件，结合了对象技术与分布式计算技术。OMG的CORBA，JAVA的RMI/EJB，Microsoft的DCOM等
5. 事物中间件（TPM）：提供支持大规模事物处理的可靠性环境，位于客户和服务器之间，完成事物管理与协调，负载均衡，失效恢复等任务。BM/BEA的TuxedoI

项目风险管理
客观性：风险是一种不以人的意志为转移，独立于人的意识之外的客观存在
偶然性：由于信息不对称，未来风险事件发生与否难以预测
相对性：风险性质会因时空各种因素变化而有所变化
社会性：风险的后果与人类社会的相关性决定了风险的社会性，具有很大的社会影响力
不确定性：发生时间的不确定性
客观性客观存在，偶然性是否发生，相对性时空变化，社会相关性，确定性何时发生
识别风险的输出：风险登记册（包括两个清单 = 已识别的风险清单 - 潜在应对措施清单）
识别风险的技术：SWOT分析
定性风险分析工具与技术 （口诀：书中进故居）：风险数据质量评估，风险种类，风险紧迫度评估，风险概率及影响评估，概率及影响矩阵，专家判断
概率和影响矩阵：对风险进行优先级排序，把风险划为低 · 中 · 高风险。风险值 = 风险发生的概率*风险发生的后果
定量风险分析工具与技术：敏感性分析，决策树分析，预期货币价值分析，龙卷风图，蒙特卡洛分析，专家判断，访谈，概率分布
只要看到 有效性 就选择 风险审计

网络安全--信息安全的基本要素
1. 机密性：确保信息不暴露给未授权的用户，实体或进程
2. 完整性：只有得到允许的人才能修改数据，并且能够判别出数据是否已篡改
3. 可用性：得到授权的实体在需要时可访问数据，可随时使用
4. 可控性：可以控制授权范围内的信息流向及行为方式
5. 可审查性：对出现的网络安全问题提供调查的依据手段
网络安全--网络和信息安全产品
1. 防火墙：通常比喻为网络安全的大门，用来鉴别什么样的数据包可以进出企业内部网。传统防火墙无法阻止和检测基于数据内容的黑客攻击和病毒入侵，同时也无法控制内部网络之间的违规行为
2. 扫描器：是入侵检测的一种，主要用来发现网络服务，网络设备和主机的漏洞，通过定期的检测与比较，发现入侵和违规行为留下来的痕迹，无法发现正在入侵的违规行为，而且它还有可能变成攻击者的工具
3. 防毒软件：可以检测，清除各种文件类型病毒，宏病毒和邮件病毒等等
4. 安全审计系统：通过独立的，对网络行为和主机操作提供全面与忠实的记录，方便用户分析与审查事故原因，就像飞机上的黑匣子

成本管理
直接成本：直接可以归属于项目工作的成本，如项目团队差旅费，工资，项目使用的物料及设备使用费等
间接成本：几个项目共同负担的项目成本所分摊给本项目的费用。如税金，额外福利和保卫费等
可变成本：随着生产量，工作量或时间而变的成本为可变成本
固定成本：不随生产量，工作量或时间变化而变化的非重复成本
机会成本：利用一定的时间或资产生产一种商品时，而失去利用这些资源生产其他最佳替代品的机会
沉没成本：由于过去的决策已经发生了的，而不能由现在或将来任何决策改变的成本

项目可行性研究报告的内容
投 - 财 - 经 - 技 - 社 - 风 - 组
  • 投资必要性：主要根据市场调查及预测结果，论证项目投资建设的必要性
  • 财务可行性：主要从项目及投资者的角度，设计合理的财务方案，从企业理财的角度进行资本预算，评价项目的财务盈利能力，进行投资决策
  • 经济可行性：主要从资源配置角度衡量项目的价值，评价项目在实现区域经济发展目标，有效配置经济资源，增加供应，创造就业，改善环境，提高人民生活等方面的效益
  • 技术可行性：主要从项目实施的技术角度，合理设计技术方案，并进行比较，选择和评价
  • 社会可行性：主要分析项目对社会的影响，包括政策体制，方针政策，经济结构，法律道德，宗教名族，妇女儿童及社会稳定性等
  • 风险因素和决策：主要对项目的市场风险，技术风险，财务风险，组织风险，法律风险，经济及社会风险等因素进行评价，制定规避风险的对策，为项目全过程的风险管理提供依据
  • 组织可行性：制定合理的项目实施进度计划，设计合理的组织机构，选择经验丰富的管理人员，建立良好的协作关系，制定合适的培训计划等，保证项目顺利执行
项目可行性研究阶段
1. 机会可行性研究，初步可行性研究，详细可行性研究，项目可行性报告的编写提交和获得批准，项目评估
2. 如果进行了项目机会研究，那么项目的初步可行性研究往往可以省去

  • 初步可行性研究可能出现的4种结果：1.肯定 - 对于较小的项目甚至可以直接 "上马"，2.肯定 - 转让详细可行性研究，进入更深入的分析研究，3.展开专题研究 - 如建立原型，演示主要功能模块，4.否定 - 项目应该 "下马"

挣值管理（EVM）是把范围，进度和资源绩效综合起来考虑，以评估项目绩效和进展方法
三个参数
  • 计划值（PV）：截至某一时刻起，应该完成的 / 可以完成的工作量的预算值
  • 实际成本（AC）：截至某一时刻起，实际的花费
  • 挣值（EV）：截至某一时刻起，已完成的 / 实际完成的工作量的预算值
4个指标
进度偏差 SV = EV - PV
  • >0，进度超前
  • < 0，进度落后
成本偏差 CV = EV - AC
  • >0，成本节约
  • < 0，成本超支
进度绩效指标 SPI = EV / PV
  • >1，进度超前
  • < 1，进度落后
成本绩效指数 CPI = EV / AC
  • >1，成本节约
  • < 1，成本超支
完工预算 BAC：所有计划值 PV 之和
完工尚需估算 ETC：剩余工作量.所需的成本
  • 经过纠正后；问题不在发生；非典型；ETC = BAC - EV
  • 不纠正；问题持续存在；典型；ETC = (BAC - EV) / CPI
完工估算 EAC：重新估算项目所需要的成本
  • 通用公式：EAC = AC + ETC
  • 通过纠正后；问题不在发生；非典型；EAC = AC + BAC - EV
  • 不纠正；问题持续存在；典型；EAC = BAC / CPI
完工偏差 VAC：完工预算 BAC 与完工估算 EAC 的差值；VAC = BAC - EAC
沟通渠道公式：N * (N - 1) / 2

案例分析的万金油
  • 看到 技术人员 出身做项目经理就要回答开发人员和管理人员所需要的技能不一样，角色定位可能不好，没有站在项目经理的角度管理项目，需要培训
  • 看到 身兼数职 就要回答一人承担两个角色的工作，可能会导致工作负荷，身心疲惫，会给全局带来不利的影响
  • 看到 新技术 就要想到风险，然后就是对大家的培训，学习，监控新技术，或者找到合适的人选从事这项工作
  • 看到 有人对项目不满 就要回答可能没有建立有效的沟通机制，缺乏有效的项目绩效管理制度，需要加强沟通
  • 看到变更的方面就一定是 变更的三个要素：书面申请 · 审批和确认 · 跟踪变更过程，严格来说，变更一定要走变更流程
  • 看到 由客户验收不通过，往往说明验收标准没有得到认可或确认，没有验收测试规范
  • 只要是与人有关的问题均可以找打 沟通 方面的答案，也就是沟通不到位
  • 看到了过来一段时间才发现问题，就说明 监控不力，不能及时发现问题
